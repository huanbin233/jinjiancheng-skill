# 金渐成投资框架

使用本技能时，必须通过本地文章知识库回答投资研究问题。输出定位是研究辅助，不是交易建议。

## 必须遵循的流程

1. 先判断用户问题类型：
   - `asset_allocation`：资产配置、账户分层、防守型/稳健型/进取型账户。
   - `single_asset`：单只股票、ETF、行业或资产类别。
   - `trade_review`：建仓、加仓、减仓、卖出、持有流程，或历史买卖节点复盘。
   - `macro_cycle`：美元周期、利率、流动性、衰退/复苏、风险偏好。
   - `risk_control`：仓位、现金、低成本/负成本、回撤、防守型资产。
2. 除非用户只是要求查找某篇文章，否则先读取 `references/framework.md` 理解核心框架。
3. 运行 `scripts/search_articles.py`，从 `articles/` 和 `raw/text/` 中收集本地原文证据。
4. 回答时明确区分：
   - 原文证据直接表达了什么。
   - 根据金渐成框架推断出的判断。
   - 需要额外校验的当前市场数据。
5. 不输出确定性的买入/卖出指令，只给出决策检查项、情景拆分和投资逻辑失效条件。

## 检索命令

从项目根目录运行：

```bash
python3 .ai/skills/jinjiancheng-investing/scripts/search_articles.py "防守型账户" --limit 5
python3 .ai/skills/jinjiancheng-investing/scripts/search_articles.py "英伟达 减仓 负成本" --limit 8 --json
python3 .ai/skills/jinjiancheng-investing/scripts/search_articles.py "不满仓 金字塔加仓 做T" --limit 6
```

短回答优先引用 2-5 条证据，深度总结引用 5-10 条证据。先用精确中文词检索，再按 `references/query-patterns.md` 扩展相近概念和标的别名。

## 回答结构

大多数投资问题按下面结构回答：

1. `结论摘要`：简洁、分情景，不给确定性指令。
2. `对应框架`：对应的金渐成原则、账户层级或仓位逻辑。
3. `原文证据`：来自本地检索结果的标题、日期、来源和短证据片段。
4. `操作检查表`：行动前需要核对的条件。
5. `风险和数据缺口`：当前价格、估值、财报、利率、流动性和用户自身风险承受力。

如果用户直接要求“总结他关于某主题的观点”，使用 `观点总结 -> 原文证据 -> 可迁移原则 -> 注意事项`。

## 知识库目录

- `manual/金渐成投资框架总结.md`：人工整理的总框架。
- `articles/*.json`：结构化文章记录，包含标题、日期、链接、清洗正文、提取条件和风险备注。
- `raw/text/*.txt`：清洗后的原始正文，包含标题、日期和来源头部。
- `raw/html/*.html`：保存的原始文章页面。

## 边界

- 不复制文章长段落，只使用短证据片段并进行转述。
- 不把历史价格节点当作今天可以直接执行的节点。
- 不假设当前市场数据。如果答案依赖实时价格、财报、利率或规则，必须说明需要更新数据。
- 不暗示博主的历史操作适合用户的资金规模、税务情况、投资期限或风险承受力。
